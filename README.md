Introduction
The Electronic Shop System is an online platform developed to help customers purchase electronic products conveniently through the internet. It also helps administrators manage products and orders efficiently.
Problem Statement
Traditional electronic stores require customers to visit physical shops. This limits accessibility and convenience. The system addresses this challenge by providing an online shopping solution.
Objectives
• Allow customers to browse products online.
• Enable shopping cart functionality.
• Support secure order placement.
• Help administrators manage products and orders.
System Features
Customer features include product browsing, cart management and checkout. Administrator features include product management, category management and order monitoring.
Technologies Used
HTML, CSS, JavaScript, PHP, MySQL, Git , GitHub and Docker were used to develop the system.
System Architecture
The system follows a three-tier architecture consisting of presentation, application and database layers.

Challenges
Session management, payment integration and deployment were some of the challenges encountered.
Future Work
Future improvements include customer reviews, email notifications and advanced search.
Conclusion
The project successfully provides an online platform for buying electronic products while improving administrative efficiency.
